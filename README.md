# 千牛 AI 客服助手 MVP（不调用淘宝接口）

这是一个 Windows 本地版 AI 电商客服助手原型。它不接入淘宝开放平台，不读取淘宝官方接口数据，而是通过当前屏幕截图 + 多模态 AI 完成：

- 识别商品详情页截图，并自动归类整理到本地知识库；
- 识别当前千牛聊天窗口截图，抽取最后一条买家消息和商品来源提示；
- 结合商品知识库和历史问答生成客服回复；
- 将回复自动粘贴到千牛输入框，默认不自动发送；
- 人工修改后的回复可以反向沉淀为知识库 FAQ。

> 第一版重点跑通闭环：商品页截图入库 → 千牛聊天截图识别 → AI 生成回复 → 自动粘贴。

## 重要说明

本项目不使用淘宝开放平台 API，不申请客服面板插件，不调用订单/聊天/商品官方接口。

它是本地桌面辅助工具，依赖卖家电脑上已打开的千牛窗口和商品详情页截图。第一版默认只自动粘贴，不自动发送，避免误回复和平台风控风险。

## v0.2 更新

- 修复从浏览器按钮触发时误截本助手页面的问题；
- 新增 `/api/windows` 窗口诊断，按进程名、窗口标题、窗口尺寸自动给千牛窗口打分；
- 识别聊天时自动寻找并激活真实千牛窗口，不再依赖“当前前台窗口”；
- 商品页采集支持选择指定窗口，或延迟 3 秒切回商品页后截图；
- 新增“当前可见历史问答入库”，可把可见聊天区的买家问题和客服回答整理为知识库。

## 安装运行

Windows 10/11 推荐：

```powershell
cd qianniu-ai-copilot
copy .env.example .env
notepad .env
.\run.ps1
```

或双击：

```bat
run.bat
```

首次运行会自动创建虚拟环境并安装依赖。

## 配置 .env

```env
OPENAI_API_KEY=sk-your-key-here
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_TEXT_MODEL=gpt-4o-mini
OPENAI_VISION_MODEL=gpt-4o-mini
```

也可以填写 OpenAI 兼容中转站：

```env
OPENAI_BASE_URL=https://你的中转站域名/v1
OPENAI_API_KEY=你的key
OPENAI_TEXT_MODEL=gpt-4o-mini
OPENAI_VISION_MODEL=gpt-4o-mini
```

## 使用流程

### 1. 商品资料自动入库

1. 打开浏览器或千牛中的商品详情页；
2. 确保商品页是当前最前面的窗口；
3. 在助手页面点击“采集当前商品页”；
4. AI 会识别商品标题、规格、价格、卖点、发货售后，并自动生成 FAQ 入库。

### 2. 识别千牛聊天

1. 打开千牛聊天窗口；
2. 点击“刷新窗口”，确认下拉框里能看到千牛候选窗口；
3. 点击“自动识别千牛聊天”，或手动选择千牛窗口后点击“识别选中窗口聊天”；
4. AI 会提取最后一条买家消息、最近聊天上下文和商品来源提示。

### 3. 生成客服回复

点击“生成 AI 回复”，系统会基于：

- 当前买家问题；
- 当前关联商品资料；
- 商品 FAQ；
- 最近聊天上下文；

生成一条可编辑的回复。

### 4. 粘贴到千牛

人工检查回复后：

1. 把光标点到千牛输入框；
2. 回到助手点击“粘贴到当前输入框”；
3. 系统会通过剪贴板粘贴回复；
4. 由人工最后确认发送。

## 数据库

本地 SQLite 数据库位于：

```text
data/copilot.db
```

主要表：

- `products`：商品知识库；
- `qa_pairs`：商品 FAQ 和人工修正问答；
- `conversations`：会话；
- `messages`：聊天消息；
- `replies`：AI 回复；
- `feedback`：人工修正记录。

## 安全边界

第一版会自动拒绝或提示人工接管以下场景：

- 退款、投诉、差评、纠纷；
- 赔偿、承诺、平台规则；
- 订单状态、地址、手机号等隐私；
- 截图无法确认的信息；
- 商品库没有依据的问题。
